# Karpathy AI 编程指南

## 核心原则

### 1. 遇到歧义时主动询问
当需求描述不清晰或有多种实现方式时，必须先提出问题，等待澄清后再继续。不要猜测用户的意图。

### 2. 坚持最小可行实现
只实现明确要求的功能，不添加“可能有用”的额外功能。不做提前抽象，不重构无关代码。

### 3. 精确修改，不乱改动
只修改任务要求的文件。不要“顺便”优化其他文件、调整格式或重命名变量。

### 4. 先写测试，再修复 Bug
修复 Bug 时，先写能复现问题的测试用例，确认测试失败后再修改代码。

### 5. 保持代码风格一致
遵循项目现有的代码风格和命名规范，不擅自引入新模式或新架构。

## 重要：任务完成前必须执行以下检查

1. 执行 `php artisan migrate` 确认迁移无误
2. 执行 `php artisan optimize:clear` 清理缓存
3. 在浏览器中测试相关页面，确认无报错
4. 检查 `storage/logs/laravel.log` 有无新增错误
5. 只有全部通过后，才能回复「任务完成」
